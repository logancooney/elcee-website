What's killing conversion on these pages

An audit of the three pages before we redesign them

The core problem: too many steps before booking

Every page buries the booking action behind multiple clicks and sections

Current Tutoring Page — 9 Sections
① HERO — big, cinematic, scroll indicator
② WHAT YOU'LL LEARN — 4-item grid
③ PRICING — 2-column price display
④ BOOKING — pick online/in-person first, then calendar loads
⑤ BULK SESSIONS — 6 bundle options in grids
⑥ THE CASE — editorial copy, no CTA
⑦ THE GUIDE — about Elcee, big image
⑧ HOW IT WORKS — 3-column process
⑨ TESTIMONIALS + FAQ
→ User must scroll past 3+ big sections just to reach the calendar
Current Booking Page — 3-Step Process
① HERO — 640px tall, scroll indicator
② STEP 1: Click a service card (studio / tutoring / mixing)
③ STEP 2: Click a mode (online / in-person, or session length)
④ STEP 3: Calendar finally appears
On mobile: 640px hero, then 3 tall cards stacked vertically with 64px padding each — that's 4+ screens of scrolling before you can book

Mobile-specific problems (no responsive design at all)

Fixed 48px padding on all sides

On a 375px wide phone, 48px padding left + right = 96px gone. Only 279px for content. Text and buttons overflow or get cramped.

Three-column grids on mobile

The service cards, bundle options, and section grids are 3-column — they likely collapse horribly or go off-screen on mobile.

56–64px vertical padding per section

Each section uses 56–64px top/bottom padding. On mobile this creates massive empty space and makes every section feel like a separate page.

No sticky CTA anywhere

There's no persistent "Book Now" button that follows the user as they scroll. The only booking entry is buried mid-page.

What a conversion-first page structure looks like

Proposed — Lean, Mobile-First Structure
① SHORT HERO — headline + price + "Book Free Session" button. Fits in one screen on mobile.
② CALENDLY EMBED — immediately visible after clicking CTA, or shown directly below hero
③ 2–3 key proof points (credibility bullets, not a wall of text)
④ Testimonials (1–2, brief)
⑤ FAQ (objection handling)
+ Sticky "Book Now" bar on mobile visible throughout scroll

Before we design the solution — one question

The Calendly embed currently requires a mode selection step first (online vs in-person for tutoring; session length for studio). Should we keep that step, or default to showing one calendar immediately?

A

Keep the selection step — just make it cleaner

Two big tappable buttons (Online / In-Person) shown immediately, then calendar. Fewer sections, same flow.

B

Skip the step — show a single calendar by default

Default to the most popular option (e.g. Online tutoring) and let users change it via a small toggle. Fastest path to booking.

C

Replace embed with a direct Calendly link

"Book Now" button opens Calendly in a new tab. Simpler, faster — no heavy iframe to load on mobile.
